1. PROJECT OVERVIEW
-------------------

The Workout Log is a console-based Java application that allows users to track their workouts, save progress to a CSV file, and reload workout history on startup. The project uses a clean separation of concerns with dedicated classes for the data model (Workout), in-memory management (WorkoutLog), user interface (WorkoutApp), and persistent storage (FileStorage).


2. CHANGES MADE SINCE TP02
--------------------------

The following changes and improvements have been identified and are being implemented for TP03:

Bug Fixes:
- Input validation: Wrapping all numeric inputs (menu choice, weight, reps, sets) in try-catch blocks with re-prompting to prevent crashes on non-numeric input
- Premature success message: Moving the "Workout added!" confirmation to only print after all fields are validated, not before
- Date format mismatch: Standardizing both the prompt and the validator to accept a single date format (MM/DD/YYYY) instead of multiple conflicting formats

New Features Added:
- User class: Implementing User.java to capture the user's name on startup, fulfilling the original TP01 proposal
- Startup flow: Adding a create/load option at startup so users can choose to start fresh or load existing workout history
- Edit workout: Adding menu option and setter methods to allow modification of existing workout entries
- Delete workout: Hooking up the existing removeWorkout() method to a new menu option with confirmation prompt
- Search/filter by date: Adding a menu option to search and display workouts matching a specific date
- Negative value rejection: Validating that weight, reps, and sets are positive numbers before accepting

Documentation Improvements:
- Upgraded all comments from basic line-level comments to meaningful Javadoc-style class and method documentation
- Updated FileStorageTest to cover new edit, delete, and search functionality

5. CURRENT APPLICATION STATE
----------------------------

What works:
- Adding workout entries via CLI menu
- Viewing all stored workouts
- Saving workout data to CSV on exit
- Loading existing workout history on startup
- CSV-safe parsing with support for commas and quotes
- Automatic backup creation and rotation
- Thread-safe file operations
