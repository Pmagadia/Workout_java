Hey team, here is the full breakdown of what we need to fix/add based on professor's TP02 feedback. I went through all the code and verified each issue. Splitting this into MUST FIX items and NEW FEATURES we need to add.


MUST FIX (Professor's Required Fixes)
--------------------------------------

1) INPUT VALIDATION - APP CRASHES ON BAD INPUT
   File: WorkoutApp.java (lines 39, 53, 56, 59)
   Problem: We use sc.nextInt() and sc.nextDouble() directly. If user types letters instead of numbers for menu choice, weight, reps, or sets, the app crashes with InputMismatchException.
   Fix: Wrap each numeric input in a loop that catches InputMismatchException and re-prompts the user. Something like "Invalid input, please enter a number."

2) "WORKOUT ADDED!" PRINTS BEFORE VALIDATION
   File: WorkoutApp.java (line 74)
   Problem: We print "Workout added!" right after creating the Workout, but we never validate the date, weight, reps, or sets at that point. Invalid workouts get silently skipped later during save. Professor specifically flagged this.
   Fix: Validate all fields BEFORE adding to the log. Only print "Workout added!" if everything passes. If something is wrong, tell the user what and re-prompt.

3) DATE FORMAT IS CONFUSING
   File: WorkoutApp.java (line 46) prompts "Enter date (MM/DD/YR):"
   File: FileStorage.java isValidDate() method (line 359-378) accepts multiple formats
   Problem: The prompt says MM/DD/YR but the validator accepts different formats. Users get confused because their date works at input time but might get rejected on save.
   Fix: Pick ONE date format (MM/DD/YYYY or MM/DD/YY) and enforce it at input time. Update the prompt to match exactly what we accept.

4) NO USER CLASS
   Problem: Our TP01 proposal said we would have a User class. It does not exist anywhere in the project.
   Fix: Create a User.java class. Minimum it should hold a username/name. The app should ask "Enter your name" on startup and tie the log to that user. Does not need to be complicated.

5) STARTUP FLOW - NO CREATE/LOAD OPTION
   Problem: The app just auto-loads from workouts.csv on startup. Professor wants explicit options.
   Fix: On startup, ask user:
     1. Create a new workout log
     2. Load an existing workout log
   Option 1 starts fresh (empty log). Option 2 loads from the CSV file.
